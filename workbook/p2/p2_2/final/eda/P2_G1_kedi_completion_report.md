**작업 파일**: `workbook/p2/p2_2/final/eda/P2_G1_kedi.ipynb`

**대상 사이트**: 한국고용정보원 고용조사분석시스템(GOMS) 주제별 통계 — https://analysis.keis.or.kr/gomsSubject.do?goIndex=3-1
**작업일**: 2026-07-12

## 1. 무엇을 지시받았는가

GOMS 주제별 통계 화면의 39개 주제(경제활동 상태 3 + 현재 일자리 18 + 근로소득 9 + 근로시간 9)를 순회하며 공식 CSV 다운로드를
자동화하고, 산출물은 별도 폴더에 정리하며, 원본은 절대 수정하지 않고, 정규화된 결과는 별도 저장하라는 지시였다.
근로소득·근로시간은 표준편차까지 받아 품질 검증에 활용하라는 요청도 포함되어 있었다.

## 2. 실제로 한 일 (시간순)

### 2.1 Gate 0 — 사이트 구조 실측
Playwright(headless Chromium)로 실제 페이지에 접속해 DOM/네트워크를 확인했다.
- 겉보기엔 Shiny 스타일 CSS(`shiny-bound-input`)를 쓰지만 실제 데이터 호출은 `POST /gomsSubjectAct.do` 단일 엔드포인트이며,
  Shiny 웹소켓 자체는 404로 죽어있다 (장식용 잔재).
- 39개 주제는 좌측 트리메뉴 그룹 `data-tt-id`: 2(경제활동 상태)·3(현재 일자리)·4(근로소득)·5(근로시간)에 정확히 분포함을 확인했다.
- "CSV 다운로드" 버튼은 실제로는 **HTML 표를 `.xls` 확장자로 감싼 파일**을 내려준다는 것을 실측으로 확인했다 (진짜 바이너리 xls/csv 아님).

### 2.2 Gate 1/2 — 자동화 중 발견하고 고친 버그 4가지 (재현 가능, 근거 첨부)

| # | 증상 | 근본 원인 | 조치 |
|---|------|----------|------|
| 1 | 두 번째 CSV 다운로드부터 계속 타임아웃 | `#cvsDownBtn` 클릭이 새 팝업(`/excel.do`)을 열거나 **기존 팝업을 재사용**하는데, 메인 페이지에서만 `download` 이벤트를 기다리고 있었음 | 메인 페이지 + 이미 열린 모든 팝업에서 동시에 `download` 이벤트를 기다리도록 수정 |
| 2 | "표준편차" 선택 후 다운로드해도 항상 "평균"과 완전히 동일한 파일 | 페이지 내장 스크립트 `callbackGOMSSubject()`가 연속형(그룹 4·5) 주제 렌더링 시 `$("#viewType2 option").eq(0).prop("selected", true)`를 **조건 없이** 실행 (범주형은 `if(old_tg != tg)` 가드가 있으나 연속형엔 없음). 즉 UI 상에서 표준편차 선택 → 검색 버튼 클릭 순간 항상 "평균"으로 강제 리셋됨 | 사이트 자체의 버그로 정상적인 방법으로는 표준편차/분산을 취득할 수 없다고 결론. **표준편차를 최종 산출물에서 제외**하고 사유를 QA 요약에 기록 |
| 3 | 연속형 18개 주제 중 다수가 서로 다른 주제인데 완전히 동일한 파일 (해시 일치) | 페이지 최초 로드시 자동 실행되는 기본 주제(`취업자의 성별 산업 분포`) 응답이 **늦게 도착**해, 이미 사용자가 선택한 다른 주제의 렌더링을 뒤늦게 덮어씀 (경쟁 상태) | 다운로드 직전 `#tresult thead`의 행 수(연속형은 1행이어야 정상, 다중 행이면 범주형 기본값이 덮어쓴 상태)를 확인하는 검증을 추가하고, 불일치 시 새 브라우저 세션으로 재시도 |
| 4 | 연속형 주제에서 `#weighted` 체크박스를 강제 클릭(`force=True`)해도 실패 | 연속형 주제에서는 사이트가 이 체크박스를 아예 `display:none`으로 숨김 (`$(".weighted").hide()`) — 연속형 통계(평균)는 항상 모집단 가중 평균으로 계산되며 가중치 옵션 자체가 없음 | 연속형 주제에서는 weighted 체크박스 조작 단계를 제거 |

이 발견 과정에서 총 3차례의 재수집이 있었다:
1. 1차 전체 실행(57건: 범주형 21 + 연속형 mean 18 + stddev 18) — 범주형 21개는 처음부터 전부 정상, 연속형 18개 중 34/36건이 버그 2·3번으로 오염된 것을 해시 비교로 발견.
2. 2차 재수집(연속형 18개, 새 브라우저 세션 + 헤더 검증) — weighted 체크박스 강제클릭 실패로 즉시 8건 실패.
3. 3차 재수집(weighted 단계 제거) — 13건 중 9건 성공, 나머지 4건(산업별·직업별(~2016) 계열, 하위 카테고리 수가 많아 응답이 느림)은 타임아웃 15초→30초로 늘려 전부 성공.

디버깅 이력 전체는 `logs/02_download_manifest_initial57_with_bugs.csv`, `logs/02b~02d_continuous_refetch_attempt*.csv`에 그대로 남겨두었다.

### 2.3 Gate 3 — 최종 39개 CSV 확정
- 범주형 21개: `raw_downloads/frequency_share/*.xls` (가중치 적용, 빈도+비중)
- 연속형(평균) 18개: `raw_downloads/mean/*.xls` (근로소득 9 + 근로시간 9)
- **표준편차 18개는 포함하지 않음** (사이트 버그로 취득 불가 — 위 표의 #2 참고)
- 최종 `02_download_manifest.csv`: **39/39 성공**, 파일 해시 전수 비교 결과 **중복 0건**

### 2.4 정규화 및 QA
- `pandas.read_html`로 파싱 (원본 마크업의 `colspan=&quot;1&quot;` 이중 이스케이프를 `"`로 치환하는 전처리 필요함을 확인 후 적용)
- 범주형 → `normalized/goms_distribution_long.{csv,parquet}` (29,160행: topic × 연도 × 하위그룹 × dimension × 빈도/비중)
- 연속형 → `normalized/goms_continuous_long.{csv,parquet}` (2,230행: topic × 연도 × dimension × 평균값)
- QA 결과 (`qa/final_qa_summary.csv`):
  - 파싱 실패 0건
  - 비중 합계 검증(90~101.5% 허용, 무응답/모름 결측 고려) 1,131건 전량 PASS
  - 연도 커버리지 39/39 PASS
  - 중복 행 0건 (범주형/연속형 모두)
  - 파일 해시 교차 중복 PASS

### 2.5 노트북(P2_G1_kedi.ipynb) 작성 및 실행
`jupyter nbconvert --execute`로 실제 실행하여 모든 셀에 진짜 출력이 담긴 상태로 저장했다 (에러 0건). 구성:
1. 작업 개요 및 2024년 프로젝트 결합 시 주의사항
2. Gate 0 탐색 결과 및 발견한 4가지 버그 설명
3. 39개 주제 레지스트리
4. 크롤러 핵심 함수 (팝업 추적, 헤더 검증 포함 최종본)
5. **라이브 데모**: 대표 주제 2개(범주형/연속형)를 지금 이 순간 실제로 재다운로드하여 크롤러가 여전히 동작함을 검증 (전체 39개 재크롤링은 15~25분이 걸리고 외부 정부 서버에 부하를 주므로 노트북을 열 때마다 재실행하지 않음 — 이미 수집된 `raw_downloads/`를 불러와 정규화·QA를 라이브로 실행)
6. 정규화 + QA 전체 파이프라인 라이브 실행

## 4. 지시사항과 달라진 점 (반드시 확인 필요)

1. **표준편차(18개)는 제공하지 못했다.** 사이트 자체의 JS 버그로 구조적으로 취득 불가능함을 소스코드 레벨에서 확인했다.
   평균(mean) 18개만 제공한다.
2. 파일 형식은 진짜 CSV가 아니라 **HTML 표를 `.xls`로 감싼 파일**이다 (사이트의 공식 동작이 그러함). 파싱은 `pandas.read_html` +
   `&quot;` 치환 전처리로 처리했다.
3. 스크린샷 57개 중 재수집된 13개 연속형 주제분은 최종 데이터와 일치하지 않는 구버전 화면이다 (데이터 자체엔 영향 없음, 감사용
   보조 자료의 한계).

## 6. 2024년 프로젝트에 결합할 때 유의사항 (원 지시사항 재확인)

GOMS는 **전년도 대졸자 약 1만8천 명의 표본조사**이며 2019년 졸업자 조사를 끝으로 잠정 중단되었다. 따라서 이 데이터는
**2024년 대학·학과 실적이 아니라 과거(2007~2019) 전국 전공계열별 노동시장 구조 기준선**으로만 사용해야 하며, 동일 전공계열의
모든 대학에 동일 값이 반복 부여되므로 독립 설명변수처럼 통계적으로 해석(p-value 등)해서는 안 된다. 인용 시 "한국고용정보원 GOMS
분석시스템의 가중 표본분석 결과"로 출처를 제한해야 한다.
